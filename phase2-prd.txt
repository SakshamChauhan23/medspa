# 4. Messaging Infrastructure

## 4.1 Twilio SMS Integration

Each clinic must be assigned a **dedicated Twilio phone number**.

Requirements:

* number provisioning per clinic
* 10DLC registration (A2P messaging compliance)
* inbound + outbound SMS support

---

### Outbound SMS Flow

```
Automation step triggered
        ↓
Message template rendered
        ↓
Twilio API send
        ↓
Delivery status stored
```

---

### Inbound SMS Flow

```
Patient sends SMS
        ↓
Twilio webhook
        ↓
API receives message
        ↓
Message stored in DB
        ↓
Automation stop conditions evaluated
```

---

### STOP Handling

When patient sends:

```
STOP
UNSUBSCRIBE
CANCEL
```

System must:

1. mark SMS opt-out
2. cancel all queued SMS jobs
3. block future messages

# 5. Automation Engine

The automation engine executes workflows.

Components:

```
Triggers
Conditions
Steps
Delays
Stop checks
```

---

## Core Concept

Each automation step is stored as a **queued job** in Redis.

Example job:

```
Send SMS → wait 2 days → send email
```

---

# 6. Workflow 1 — Botox Reactivation

This is the **highest priority workflow**.

Purpose:

Reactivate Botox clients who have not returned.

---

## Trigger Detection

Daily scheduled scan.

Condition:

```
treatment_type = botox
AND last_treatment_date >= 90 days
AND no upcoming appointment
```

---

## Workflow Sequence

```
Day 0
Send SMS reminder

Day 2
Send Email reminder

Day 5
Send Final SMS
```

---

## Stop Conditions

Before every step check:

```
patient booked appointment
OR patient replied STOP
OR patient opted out
```

If true:

```
cancel workflow
```

---

# 7. Workflow 2 — Appointment Reminder

Purpose:

Reduce appointment no-shows.

---

## Trigger

```
appointment_time < 24 hours
```

---

## Workflow

```
24h before appointment
Send reminder SMS

Wait 20 hours

Send confirmation request
"Reply YES to confirm"
```

---

## YES/NO Parsing

Inbound Twilio webhook must parse:

```
YES → mark appointment confirmed
NO → send reschedule link
```

---

## Escalation

If no response:

```
send follow-up reminder
```

---

# 8. Workflow 3 — Cancellation Recovery

Purpose:

Refill cancelled appointment slots.

---

## Trigger

Event:

```
appointment_cancelled
```

---

## Workflow

```
Find patients on waitlist
↓
Send SMS
"Slot available today at 3pm. Reply YES to book."
↓
First responder gets slot
```

---

# 9. Workflow 4 — Membership Retention

Purpose:

Prevent membership churn.

---

## Trigger

```
membership_expiry approaching
```

---

## Workflow

```
30 days before expiry → reminder email

7 days before expiry → SMS reminder

Final day → final SMS
```
