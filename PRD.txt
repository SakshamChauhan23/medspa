# Product Requirements Document (PRD)

## Med Spa Revenue Rebooking System

---

# 1. Product Overview

### Product Name

Med Spa Revenue Rebooking System

### Product Category

Revenue Recovery Infrastructure for Med Spas

### Product Goal

Increase clinic revenue by:

1. Reactivating dormant patients
2. Completing unfinished treatment packages
3. Preventing appointment cancellations and no-shows
4. Improving patient retention through automated follow-ups

The system integrates with **existing clinic booking software (PMS)** and automates **treatment-cycle communication workflows** via SMS and email.

---

# 2. Problem Statement

Most med spas lose significant revenue because:

1. Clients forget to return for treatment cycles
2. Treatment packages remain incomplete
3. Appointments get cancelled or missed
4. Staff cannot consistently follow up with patients

Clinics focus heavily on **new patient acquisition** while ignoring **existing patient retention**, causing large amounts of **idle revenue inside their database**.

This system solves the problem by **automating rebooking and retention workflows**.

# 8. Automation Workflow Diagrams

Below are the **core workflows that must be implemented in MVP**.

---

# Workflow 1

## Botox Reactivation Automation

Trigger

Patient overdue for Botox

Condition

```
last_botox_date ≥ 90 days
AND no upcoming appointment
```

Automation Flow

```
Start
 ↓
Check Botox overdue
 ↓
Send SMS Reminder
 ↓
Wait 2 days
 ↓
Send Email Reminder
 ↓
Wait 3 days
 ↓
Send Final SMS
 ↓
Stop
```

Stop conditions

```
Patient books appointment
OR patient replies
```

---

# Workflow 2

## Laser Package Completion

Trigger

Incomplete package

Condition

```
sessions_remaining > 0
AND last_session > 14 days
```

Automation Flow

```
Start
 ↓
Package incomplete detected
 ↓
Send SMS reminder
 ↓
Wait 3 days
 ↓
Send email follow-up
 ↓
Wait 5 days
 ↓
Send final reminder
```

Stop

```
Appointment booked
OR reply received
```

---

# Workflow 3

## Membership Retention

Trigger

Membership expiry approaching

Condition

```
membership_expiry < 30 days
```

Flow

```
Start
 ↓
Send renewal reminder
 ↓
Wait 7 days
 ↓
Send follow-up reminder
 ↓
Wait 10 days
 ↓
Send final reminder
```

Stop

```
Membership renewed
```

---

# Workflow 4

## Appointment Reminder

Trigger

Appointment scheduled

Condition

```
appointment_time < 24 hours
```

Flow

```
Start
 ↓
Send 24-hour reminder
 ↓
Wait 20 hours
 ↓
Send confirmation request
 ↓
Patient confirms
 ↓
Stop
```

If not confirmed

```
Send follow-up reminder
```

---

# Workflow 5

## Cancellation Recovery

Trigger

Appointment cancelled

Flow

```
Start
 ↓
Detect cancellation
 ↓
Check waitlist
 ↓
Send slot availability SMS
 ↓
First patient books
 ↓
Slot filled
 ↓
Stop
```

---

# Workflow 6

## No-Show Reduction

Trigger

Patient does not confirm appointment

Flow

```
Start
 ↓
Appointment booked
 ↓
24h reminder sent
 ↓
No confirmation
 ↓
Send confirmation request
 ↓
If confirmed → stop
If not confirmed → send final reminder
```
